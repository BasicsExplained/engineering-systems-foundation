# Checkpoint 07 — Database and Persistence

## Goal

Understand why databases exist and what persistence means.

## Reading

If data is stored only in program memory, it disappears when the program stops. A database stores data so it can survive restarts and be queried later.

## Your task

1. Store readings in memory first.
2. Restart the backend and observe data loss.
3. Store readings in a local database.
4. Restart again and show data remains.
5. Run at least one query.

## Required evidence

- Before/after restart proof.
- Screenshot or output showing database records.
- One query result.
- Explanation of memory vs database.

## Mentor questions

- What happened after restart?
- What is a table?
- What is a row?
- Why is timestamp useful?
