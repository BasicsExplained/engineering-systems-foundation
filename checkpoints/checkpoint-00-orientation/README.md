# Checkpoint 00 — Orientation

## Goal

Understand how this course works and set up your progress-sharing folder before starting technical work.

## Instruction

Do these tasks in order:

1. Read the homepage once.
2. Read Progress Sharing and create your Google Drive folder.
3. Download or copy the provided templates.
4. Create your Progress Tracker Google Sheet.
5. Create your Project Journal Google Doc.
6. Share the main Google Drive folder with the mentor.
7. Read Course Rules.
8. Read No-AI Policy.
9. Read How to Search and Debug.
10. Write a short answer: What do you think an engineered system is?

Do not start coding in this checkpoint.

## Example

Example of an engineered system:

```text
water tank -> water level sensor -> level data -> program checks level -> dashboard shows status -> engineer checks logs if level looks wrong
```

This is only an example. You do not need to choose a water tank.

## Submit

Send these items to the mentor:

1. Google Drive folder link.
2. Screenshot of your folder structure.
3. Screenshot or link to your Progress Tracker.
4. Screenshot or link to your Project Journal.
5. One paragraph explaining the course goal in your own words.
6. One paragraph explaining what Signal-to-Cloud means.
7. One question you still have.

## Done when

This checkpoint is complete when:

- your Google Drive folder is created and shared
- your Progress Tracker exists
- your Project Journal exists
- you can explain why the course uses checkpoints
- you can explain why evidence is required
- you can explain why copied answers are not allowed
- you can explain what Signal-to-Cloud means

## Mentor review

The mentor may ask:

- Where will you submit checkpoint evidence?
- What is the difference between the course website, Google Drive, and GitHub?
- What is the difference between finishing a task and understanding a system?
- Why should every checkpoint include evidence?
- What does Signal-to-Cloud mean in simple words?
