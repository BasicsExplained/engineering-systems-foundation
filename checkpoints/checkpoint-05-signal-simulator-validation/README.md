# Checkpoint 05 — Signal Simulator and Validation

## Goal

Create simulated signal data and validate normal, warning, critical, and invalid values.

## Reading

Real systems receive messy data. Sensors can send missing, delayed, impossible, or noisy values. Engineering systems must validate input before trusting it.

## Your task

1. Choose at least three signals for your system.
2. Define normal, warning, and critical ranges.
3. Generate changing simulated values.
4. Generate bad values sometimes.
5. Reject impossible values and warn for risky values.

## Required evidence

- Signal definitions and range justification.
- Sample simulator output.
- Log showing invalid data rejected.
- Explanation of normal vs abnormal vs impossible.

## Mentor questions

- Why did you choose these ranges?
- What is invalid data?
- What should be rejected versus only warned?
- How can bad data affect real systems?
