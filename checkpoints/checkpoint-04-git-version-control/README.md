# Checkpoint 04 — Git and Version Control

## Goal

Understand Git as version control and GitHub as only one cloud hosting service for Git repositories.

## Reading

Git tracks changes in your project over time. GitHub hosts Git repositories online, but Git can work without GitHub. Other hosting services include GitLab, Bitbucket, and self-hosted Git servers.

## Your task

1. Initialize Git in your project.
2. Make at least five meaningful commits.
3. Intentionally break something and commit the broken state.
4. Fix it and commit again.
5. Push to GitHub.
6. Explain Git vs GitHub.

## Required evidence

- Screenshot of git log.
- Repository link.
- Explanation of commit, branch, remote.
- One Git error or confusion you faced.

## Mentor questions

- Can Git work without GitHub?
- What is a commit?
- What is a remote?
- Why should commits be meaningful?
