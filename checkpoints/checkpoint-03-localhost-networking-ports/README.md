# Checkpoint 03 — Localhost, Networking, and Ports

## Goal

Understand how one program becomes reachable through a network address and a port.

## Reading

`localhost` means this computer. `127.0.0.1` also points to this computer. A LAN IP address points to your computer from another device on the same network. A port identifies which program/service should receive the request.

## Your task

1. Run a local server.
2. Open it using `localhost`.
3. Open it using `127.0.0.1`.
4. Find your laptop's LAN IP.
5. Try opening the server from a phone on the same Wi-Fi.
6. Intentionally create a port conflict.
7. Explain what failed and why.

## Required evidence

- Laptop browser screenshot.
- Phone browser screenshot, or explanation if blocked.
- LAN IP used.
- Port conflict screenshot/log.
- Search trail.

## Mentor questions

- Why does localhost on phone not mean your laptop?
- What is a port?
- What does connection refused mean?
- What can block network access?
