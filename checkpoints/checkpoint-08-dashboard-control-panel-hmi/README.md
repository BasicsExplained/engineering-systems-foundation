# Checkpoint 08 — Dashboard and Control Panel as HMI

## Goal

Understand the dashboard as a human-machine interface, not just a website.

## Reading

A dashboard helps a human understand system state and make decisions. A control panel lets the user manipulate inputs and observe system response.

## Your task

Create a simple dashboard/control panel showing:

- latest reading
- status: normal, warning, critical, offline
- last 10 readings
- manual input or simulation controls
- one chart or table

## Required evidence

- Screenshot of dashboard.
- Explanation of each displayed field.
- One frontend/backend connection issue and fix.
- Explanation of the decision a user should make from the dashboard.

## Mentor questions

- What is the dashboard telling the user?
- What can the user manipulate?
- What happens if backend is down?
- Why is this an HMI?
