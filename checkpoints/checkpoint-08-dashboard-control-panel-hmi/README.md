# Checkpoint 08 — Dashboard and Control Panel as HMI

## Goal

Understand the dashboard as a human-machine interface, not just a website.

## Instruction

Do these tasks in order:

1. Create a simple dashboard or control panel.
2. Show the latest reading.
3. Show status: normal, warning, critical, or offline.
4. Show the last 10 readings.
5. Add manual input or simulation controls.
6. Add one chart or table.
7. Break the frontend/backend connection once and fix it.
8. Explain what decision a user can make from the dashboard.

## Example

Example dashboard fields:

```text
Latest temperature: 28.5
Latest humidity: 61
System status: normal
Last update: 10:30 AM
```

Example user decision:

```text
If status becomes critical, the user should inspect the system or stop the process.
```

## Submit

Send these items to the mentor:

1. Screenshot of the dashboard.
2. Explanation of each displayed field.
3. Screenshot or notes from one frontend/backend connection issue.
4. Explanation of the fix.
5. Explanation of the decision a user should make from the dashboard.

## Done when

This checkpoint is complete when:

- the dashboard displays live or recent system data
- the user can understand system status
- one frontend/backend issue was identified and fixed
- you can explain why this is an HMI

## Mentor review

The mentor may ask:

- What is the dashboard telling the user?
- What can the user manipulate?
- What happens if the backend is down?
- Why is this an HMI?
- What decision should the user make from your screen?
