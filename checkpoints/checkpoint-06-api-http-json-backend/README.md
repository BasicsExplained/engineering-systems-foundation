# Checkpoint 06 — API, HTTP, JSON, Backend

## Goal

Understand how programs communicate using HTTP APIs and JSON.

## Reading

An API is a controlled communication point. A client sends a request. A server processes it and returns a response. JSON is a common data format for exchanging structured data.

## Your task

Create a simple backend for your system with endpoints like:

```text
GET /health
POST /reading
GET /latest
GET /history
```

Send simulator readings to the backend.

## Required evidence

- Screenshot of backend running.
- Successful API request.
- Failed API request and explanation.
- Explanation of GET, POST, endpoint, request, response, JSON.

## Mentor questions

- Why is POST used for sending a reading?
- What is JSON?
- What happens when invalid data is sent?
- What is the backend responsible for?
